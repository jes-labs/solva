# Solva Orchestrator (Go)

The orchestrator runs Solva's proof cycle: fetch signed reserves, load
liabilities, call the Rust prover, publish the proof on Stellar, then persist
and cache the result. It exposes the REST API and the scheduler.

## Clean architecture

Dependencies point inward. An outer layer may import an inner one; an inner
layer never imports an outer one.

```
controller/http ─┐
infrastructure  ─┼─► usecase ─► entity
repo            ─┘
```

- `internal/entity`: pure domain structs (`Proof`, `ReserveSnapshot`,
  `Tenant`, `InclusionRef`). Imports only the standard library. Money is a
  decimal string of integer minor units, never a float.
- `internal/usecase`: business logic (`Cycle`, `Query`) and every port
  interface (`ProverClient`, `StellarPublisher`, `BankAdapter`, `ProofRepo`,
  `ReserveRepo`, `Cache`). Imports only `entity`.
- `internal/infrastructure`: adapters that implement the ports: `grpc`
  (prover client), `stellar` (Soroban publisher), `banks` (Open Banking
  adapter plus ECDSA verification), `scheduler` (cron trigger).
- `internal/repo`: Postgres via pgx and Redis via go-redis. Typed queries are
  generated by sqlc from `query/*.sql` into this package.
- `internal/controller/http`: chi router and handlers. Handlers call use
  cases through their interfaces and never touch infrastructure directly.

## Crypto boundary

The orchestrator performs no hashing that must match the circuit. Poseidon2
lives only in the Rust prover and the Soroban contract. The one piece of real
cryptography here is standard ECDSA verification of bank balance responses, in
`internal/infrastructure/banks/ecdsa.go`.

## Generated code

- gRPC stubs: `just proto` generates `internal/infrastructure/grpc/proverpb`
  from `/proto/prover.proto`.
- SQL: `just sqlc` generates typed queries into `internal/repo` from
  `sqlc.yaml`. Generated code is not committed; regenerate from source.

## Run

```
go run ./cmd/app
```

Configuration comes from the environment. See `internal/config/config.go` for
keys and defaults. Postgres and Redis must be reachable at boot.
